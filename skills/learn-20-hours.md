# Learn Anything in 20 Hours

Build the 20% that drives 80% of results. Web search required for current best resources.

## Step 1 — Define the Learning Goal

Before building the plan, establish:
- **Topic**: [What exactly needs to be learned]
- **Purpose**: [Why — BD call prep / investment thesis / skill gap / personal interest]
- **Deadline**: [When does CB need to be functional with this?]
- **Starting level**: [Zero / some exposure / intermediate]
- **Output goal**: [Can explain it / can apply it / can teach it / can decide with it]

## Step 2 — Identify the 20% Core

Research the topic and identify:
- The 3–5 core concepts that underpin everything else
- The most common practitioner mistakes (what NOT to learn first)
- The fastest path from zero to functional

Flag: "If CB only learns these [X] things, they can handle 80% of real-world situations with this topic."

## Step 3 — Build the 20-Hour Session Plan

Structure as 8–10 sessions of 2–3 hours each. Each session:

| Session | Focus | Key Concepts | Resource | Review Checkpoint |
|---|---|---|---|---|
| 1 | Foundation | Core vocabulary, mental model | [Best intro resource] | Can you explain the core concept in plain English? |
| 2 | Mechanics | How it actually works | [Second resource] | Can you walk through a basic example? |
| 3–5 | Application | Real-world use cases, patterns | [Case studies / examples] | Can you apply it to CB's specific context? |
| 6–7 | Depth | The 20% that separates practitioners | [Advanced source] | Can you identify what you don't know yet? |
| 8 | Edge cases | What breaks the model | [Practitioner resource] | Can you spot when the rules don't apply? |
| 9 | Integration | Connect to CB's existing knowledge | CB's context | Can you explain how this changes your current approach? |
| 10 | Review + Teach | Feynman pass — explain it cold | — | Can CB teach it in 5 minutes without notes? |

## Step 4 — Resource Stack (web search required)

For the topic, find:
1. **Best book** — foundational, time-tested
2. **Best video/course** — fastest way to see it in action
3. **Best practitioner source** — someone who does this, not just teaches it
4. **Best case study** — real-world application CB can analyze
5. **Best community/forum** — where practitioners discuss edge cases

Flag source quality and estimated time investment for each.

## Step 5 — CB-Specific Hooks

Connect the topic to CB's existing context:
- How does this connect to energy/power industry knowledge?
- How does this connect to passive income / financial goals?
- How does this connect to business building / BD skills?
- Which mental models from CB's library apply here?

## Hard Rules
- Never build a 20-hour plan without web-searching current best resources — outdated materials waste the hours
- Always anchor to CB's purpose — learning for its own sake is not the goal
- Flag if the topic requires more than 20 hours to reach CB's stated output goal
