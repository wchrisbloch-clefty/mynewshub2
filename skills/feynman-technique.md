# Feynman Technique — Deep Mastery Loop

TRIGGER-ONLY. This is a teaching loop, not a lecture. It runs until CB owns the concept.

## When to Use (and When NOT to)

**Use when:**
- CB explicitly triggers `feynman [topic]`
- Spine flags a blind spot that requires genuine mastery before acting
- Preparing for a high-stakes meeting where CB must explain or defend a concept
- CB has hit Level 3 on the Learning Ladder and wants to verify real understanding
- Before a Council session on a topic CB hasn't fully internalized

**Do NOT use when:**
- CB asks a factual question → answer directly
- CB needs a quick definition → give it
- CB is exploring, not mastering → follow the thread
- Any situation where a direct answer is faster and sufficient

## The Four-Pass Loop

### Pass 1 — Plain English Explanation

Explain [topic] as if teaching a smart 12-year-old. No jargon. No assumptions.

Format:
> "Here's [topic] in plain English: [explanation]"
> "The core idea is: [one sentence]"
> "A simple example: [concrete, real-world illustration]"

Then ask: "Can you explain this back to me in your own words — without looking at what I just wrote?"

### Pass 2 — Gap Detection

After CB's explanation, identify:
- What was accurate
- What was missing
- What was subtly wrong (the dangerous partial understanding)
- What CB assumed was obvious but isn't

Format:
> "You've got [X] right. Here's what's missing: [gap]"
> "Watch out for this: [common misconception CB showed]"

Re-teach the gap only. Don't re-explain everything.

### Pass 3 — Application Test

Give CB a real scenario to work through:

> "Here's a situation: [specific scenario tied to CB's world — energy, finance, BD, or current goal]. How does [topic] apply here?"

Assess the response:
- Can CB apply the concept correctly?
- Does CB know when the concept doesn't apply?
- Can CB spot the edge case?

### Pass 4 — Cold Explain

Final test — no prompts, no notes:

> "Explain [topic] to me like I've never heard of it. You have 3 minutes. Go."

If CB can do this clearly and correctly: **mastery confirmed.**
If not: identify the remaining gap, run a targeted mini-pass, retest.

## Mastery Confirmation

When CB passes Pass 4:

> "Mastery confirmed: [topic] — [date]"
> "CB can now: [what this unlocks — specific to CB's context]"
> "Feynman-ready: YES — add to monthly state"

Flag for Spine monthly state: `Feynman ready: [topic] — [date]`

## Hard Rules
- Never run this loop for a casual question — direct answers are faster and better in those cases
- Never skip to Pass 4 — the gaps only surface through the full loop
- Never confirm mastery just because CB sounds confident — test it
- Always tie the application test (Pass 3) to CB's actual world: energy, finance, BD, or current project
- One topic per session — depth over breadth
